# Faction system, slice 1: the Church + the faction framework
*Implementation plan, July 2026. Status: **implemented — July 2026.** Distilled from a design session; supersedes the flavor-first sketch in [factions.md](../factions.md) where they conflict. The docs (CLAUDE.md, design-doc.md, factions.md, building-effects.md) now carry the built truth; this plan is kept as the design record.*

## Context

First slice of [factions.md](../factions.md), evolved through design discussion well past the doc's flavor-first sketch. Decisions, in order:
1. **Profile + ladder, no signature chain yet** (chain is its own slice; capstone wants architects).
2. **Patron admission gates commissions**: the pool starts empty — Chapel (400ƒ) admits the Church, each Palazzo (900ƒ) installs the next noble house (Medici → Strozzi → Pazzi, table order), Cathedral (1500ƒ) also admits the Church and gates its upper rungs.
3. **A faction banner UI** (Civ 6 diplomacy-banner style): clickable crest per admitted patron → card with favor, standing, effects. Overturns factions.md's "no faction panel" non-goal — deliberately.
4. **Favor is a real relationship meter** — overturns the "no relationship meters" non-negotiable (CLAUDE.md + design principle 4), deliberately. Moves on decisions *and* neglect; no time decay (still banned).
5. **Neglect's supporting cast**: since missing an offer costs favor, arrivals must be unmissable and rare — a Civ-trade-request-style arrival popup, offer rate roughly halved, rewards buffed to compensate.
6. **Favor is a 0–100% scale**: rungs unlock by *being above* favor levels, not by cumulative works; slight negative effects below one threshold, a rare severe tier below a lower one — including a one-time denunciation prestige hit, the design's single sanctioned citywide consequence.

The guild requesters get removed (doc-slated). Favor/rung/tier mechanics are **requester-generic** (houses ride the same code free); Church-specific parts: chapel admission, cathedral rung-gate, church title pools. Family taste jitter, seed-shuffled install order, rivalry pairs, chains: future slices.

## Favor model (0–100 per faction)

- **Persisted `favor: Record<string, number>`**; reads default to `FAVOR_START = 50` (neutral), so admission needs no init event.
- **Moves** (clamped 0–100): **+8** per completed work (`FAVOR_PER_WORK`); **−5** per declined *or* expired open offer (`FAVOR_SLIGHT`). Nothing else; no drift.
- **Rungs by current level**: ≥60 / ≥75 / ≥90 (`FAVOR_RUNGS`) → grandeur `FAVOR_GRANDEUR = [1, 1.3, 1.6, 2]` multiplying duration, florins, prestige of that faction's offers. Church rungs 2–3 additionally require a standing cathedral (effective rung caps at 1 without; favor itself is untouched).
- **Cooled, favor < 35**: the faction's offers skip half the time (one extra rng draw after the requester pick — new draw order pinned in the check) and are forced to rung 0.
- **Affronted, favor < 15**: offers skip 75% of the time, rung 0 — near-silence, but every rare offer is the recovery path. **Crossing into affronted fires a one-time denunciation**: −15 city prestige (`DENOUNCE_PRESTIGE`, clamped at 0) + an alert card ("House Pazzi denounces your patronage"). Re-arms only after favor recovers above 15.
- **Rarity check**: from 50, seven consecutive slights with zero completions to reach affronted — rare and deliberate, as specified.
- **Pacing rebalance**: `COMMISSION_OFFER_CHANCE` 0.15 → **0.08**/month (~one offer/year); rewards compensate — `FLORINS_PER_PRESTIGE` 25 → **40**, new `COMMISSION_PRESTIGE_SCALE = 1.5` on minted base prestige (constants.ts "quality ≈ 1..20" display comment updated). All knobs, all starting points.

## Changes

1. **`app/game/constants.ts`** — factions block: `FAVOR_START`, `FAVOR_PER_WORK`, `FAVOR_SLIGHT`, `FAVOR_RUNGS = [60, 75, 90]`, `FAVOR_GRANDEUR`, `FAVOR_COOLED = 35`, `FAVOR_AFFRONTED = 15`, `COOLED_SKIP_CHANCE = 0.5`, `AFFRONTED_SKIP_CHANCE = 0.75`, `DENOUNCE_PRESTIGE = 15`, `COMMISSION_PRESTIGE_SCALE`; retune `COMMISSION_OFFER_CHANCE`, `FLORINS_PER_PRESTIGE`.
2. **`app/game/commissions.ts`** — drop guild entries (`RewardMix` loses `"mixed"`); `requesterPool(tiles)` (Church if chapel/cathedral origin stands + first `min(palazzoCount, 3)` houses); `favorOf(favor, name)` (default 50) + `favorRung(name, value, tiles)` (thresholds + cathedral cap) + `favorTier` (affronted/cooled/neutral); `maybeOfferCommission(commissions, artists, currentTick, rng, tiles, favor)`: empty pool → null before rng; requester pick from pool; tier skip-draw; grandeur × duration/florins/prestige; Church titles from `CHURCH_TITLES[type]` (bronze keeps `BRONZE_TITLES`). `reconcileCommissions` also returns `expiredRequesters: string[]`.
3. **`app/game/artists.ts`** — `CHURCH_TITLES: Record<ArtistType, string[]>` beside `TITLES` (frescos/altarpieces/devotional; saints/pietàs; chapel designs).
4. **`app/game/tick.ts`** — `TickSnapshot` gains `favor`; `TickTransition` gains `favor` + `denounced: string[]`; apply +8 per completion / −5 per expiry with clamps; detect ≥15 → <15 crossings for the denunciation (prestige hit inside the transition, clamped ≥0).
5. **`app/stores/useGameStore.ts`** — `favor` in `GameState`/`createInitialState`/`tick()` mapping/`partialize`. `declineCommission(commissionId)`: drop open offer (no-op if assigned/missing), −5 favor, same crossing check. Transients (never persisted, `razeTarget` pattern): `offerAlert: string | null` (set when a tick adds a new open offer id; overwrite on rare doubles) and `denounceAlert: string | null` (requester name, from tick's `denounced` or a declining crossing). `saveMigration.ts` → `SAVE_VERSION = 8`: favor seeded as `clamp(50 + 8 × per-requester artwork count, 0, 100)` — old saves keep earned standing.
6. **UI** — new **`app/game/ui/FactionBanner.tsx`** (mounted in `GameHUD.tsx`): horizontal crest stack fixed top-right below the settings button in the top bar, one banner per admitted patron; click toggles that patron's card (local one-open state): name, favor %, standing label (Affronted/Cooled/Neutral/rung names), next-rung threshold, cathedral-gate hint, recovery hint when below 35. New **`app/game/ui/OfferAlert.tsx`** (mounted in GameHUD, handed `onView={() => setOpenPanel("commissions")}` since panel state is local there): renders the `offerAlert` commission as a persistent non-blocking arrival card — requester, title, rewards, required type, **View** / **Later** — and renders `denounceAlert` as its darker sibling card; renders nothing if the offer meanwhile expired/assigned. **`CommissionsPanel.tsx`**: Decline button on offers → `declineCommission`; "— Nth work" beside the requester (per-requester artwork count + 1, shown when ≥1 prior work — cumulative flavor, now decoupled from the meter; `ordinal()` in `ui/format.ts`); contextual empty state ("Build a Chapel — the Church will bring commissions.") when the pool is empty.
7. **Checks** — `commissions.check.ts` reworked: pool gating (always-null when empty, per-palazzo installs), rung thresholds + cathedral cap, grandeur math, cooled/affronted skip draws + rung-0 forcing, new draw order pinned, guild fallout (unskewed-base fixtures re-derive through church/noble skew), reward expectations re-pinned against the new knobs. `tick.check.ts`: favor deltas, clamps, denounce crossing + prestige clamp, snapshot fixtures gain `favor: {}`. `saveMigration.check.ts`: v8 seeding.
8. **Docs (truthfulness pass)** — `CLAUDE.md`: rewrite the "no relationship meters" non-negotiable to the new rule (0–100 per-faction favor moved by decisions + neglect, faction-scoped consequences except the one-time denunciation, no decay, not a top-bar resource) + current-state note. `design-doc.md`: principle 4 rewrite; Requesters section (admission, favor scale, tiers, decline, slower-but-richer pacing, arrival popup); chapel/palazzo entries; UI layout note (banner + popup). `factions.md`: rewrite the favor-ladder section to the 0–100 model, banner UI, prune non-goals (panel, meter-down), keep bans (decay, lockouts, fourth resource). `building-effects.md`: mark requester-pool shaping built.
